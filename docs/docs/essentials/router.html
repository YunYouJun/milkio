Router ​

Router ​

在 快速开始 章节中，你肯定体验到了路由的"魔法"。在 Milkio 中，你几乎不需要亲自编写路由的代码。Milkio 会自动阅读 /src/apps 目录下的文件，并为你所有的 API 自动生成路由。

In the Quick Start section, you must have experienced the "magic" of routing. In Milkio, you hardly need to write the routing code yourself. Milkio will automatically read the files in the /src/apps directory and generate routes for all your APIs.

补充路由逻辑 ​

Supplement routing logic ​

在几乎所有情况下，你都只会需要 Milkio 为你自动生成的路由。但或许会有一些特殊情况，你需要自定义你的路由。此时，你可以编辑 /src/router.ts 文件。其内容类似如下这样：

In almost all cases, you will only need the routes automatically generated by Milkio for you. However, there may be some special cases where you need to customize your routes. In such situations, you can edit the /src/router.ts file. Its content will be similar to the following:

ts
export const routerHandler = async (path: string, fullurl: URL) => {
  // ...
  return false;
};
ts
export const routerHandler = async (path: string, fullurl: URL) => {
  // ...
  return false;
};

你可以编辑此方法的内容，以补充 Milkio 自动生成的路由之外的期望路由匹配逻辑。你可以通过返回一个已存在的 API 地址，它将被调用。如果你返回 false，将返回一个 NOT_FOUND 的失败信息。

You can edit the content of this method to supplement the expected route matching logic outside of the routes automatically generated by Milkio. You can return an existing API address that will be called. If you return false, a NOT_FOUND failure message will be returned.

同时，此方法接受两个参数：path 和 fullurl。path 表示请求路径的后缀（删除了域名和 忽略的路径层级），出于性能考虑，如果 URL 中包含参数，这些参数不会被删除，仍然保留在 path 中，例如 foo/bar?hello=world。fullurl 是完整的 URL 对象，忽略的路径层级 不会被删除。

Additionally, this method accepts two parameters: path and fullurl. path represents the suffix of the request path (with the domain and ignored path levels removed). For performance reasons, if the URL contains parameters, these parameters will not be removed and will still be preserved in the path, for example, foo/bar?hello=world. fullurl is the complete URL object, and the ignored path levels will not be removed.

INFO

请注意，此方法的优先级永远低于 Milkio 自动生成的路由，这意味着你无法禁止某个 API 的访问。这样设计的目的是为了确保你尽量只在此处编写路由匹配规则。如果你需要对某个 API 进行授权才能访问，请使用中间件功能。

INFO

Please note that the priority of this method is always lower than the automatically generated routes by Milkio, which means you cannot prohibit access to a specific API. This design is intended to ensure that you only write routing matching rules here as much as possible. If you need to authorize access to a specific API, please use the middleware function.
