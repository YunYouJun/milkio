Get Started ​

所有的 Milkio 工程，都拥有一个自己的客户端。其他使用 TypeScript 的工程可以使用这个客户端来轻松地调用你的 Milkio 工程，享有完善的类型检查、自动补全，真正的类型即是文档。

All Milkio projects have their own client. which can be used by other projects using TypeScript to easily call your Milkio project, enjoying comprehensive type checking, auto-completion, and the true type being the documentation.

ts
const result = await client.execute("hello-world/say", "🥛 Milkio")
ts
const result = await client.execute("hello-world/say", "🥛 Milkio")

尝试一下 ​

Try it ​

我们假设，你已经有一个想要调用 Milkio 的工程，无论它是 Vue、React、Electron、React Native、Mini Program 或者任何，甚至是另一个后端工程都可以，只要它们支持使用 TypeScript 语言（即使它们的代码不是由 TypeScript 编写的）。如果你还没有，我们推荐你使用 Vite。

We assume that you already have a project in mind that you want to integrate Milkio into, whether it's Vue, React, Electron, React Native, Mini Program, or any other project, as long as they support TypeScript (even if their code is not written in TypeScript). If you haven't already, we recommend using Vite.

当你准备好之后，现在我们假设，它们恰好在同一个目录下，大致的目录结构就像下面这样：

When you are ready, let's assume that they are conveniently located in the same directory, and the approximate directory structure would look like this:

sh
workspace
├─ a-milkio
│  ├─ ...
│  └─ package.json
└─ a-web-app
   ├─ ...
   └─ package.json
sh
workspace
├─ a-milkio
│  ├─ ...
│  └─ package.json
└─ a-web-app
   ├─ ...
   └─ package.json

现在，我们在 VS Code 打开你的 web 工程，并通过点击 VS Code 的 文件 -> 将文件夹添加到工作区.. 按钮，来打开你的 Milkio 工程。这样，你就在你的 VS Code 中同时打开了两个工程。

Now, we open your web project in VS Code and add your Milkio project to the workspace by clicking on File -> Add Folder to Workspace.. in VS Code. This way, you have both projects open in your VS Code simultaneously.

在 VS Code 中，选择 终端 -> 新建终端，并选择你的 web 工程，之后在终端中，安装你 Milkio 工程中的客户端包：

In VS Code, select Terminal -> New Terminal, and choose your web project. Then, in the terminal, install the client package for your Milkio project.

sh
bun i ../a-milkio/packages/client
sh
bun i ../a-milkio/packages/client

在你的 web 工程中，在你喜欢的位置，新建一个 client.ts 文件，并在其中写入下面的代码：

In your web project, create a new client.ts file at your preferred location and write the following code in it:

ts
import { createClient } from 'client'

export const client = createClient({
  baseUrl: 'http://localhost:9000/',
  storage: localStorage,
  async handler(url, body, headers) {
    return await (await fetch(url, { body, headers })).text()
  }
})
ts
import { createClient } from 'client'

export const client = createClient({
  baseUrl: 'http://localhost:9000/',
  storage: localStorage,
  async handler(url, body, headers) {
    return await (await fetch(url, { method: 'POST', body, headers })).text()
  }
})

上面的代码假设了你的 web 工程运行在浏览器里，所以它使用了 localStorage 和 fetch，如果你运行在没有 localStorage 或者 fetch 的环境下，你可以自由地替换它，只要符合 TypeScript 类型即可。

The above code assumes that your web project is running in a browser, so it uses localStorage and fetch. If you are running in an environment without localStorage or fetch, you are free to replace them as long as they conform to the TypeScript types.

试试来通过客户端，来执行一下你的 Milkio 工程 吧！UwU

Please try to execute your Milkio project through the client! UwU

ts
const result = await client.execute("hello-world/say", "🥛 Milkio")
console.log(result)
ts
const result = await client.execute("hello-world/say", "🥛 Milkio")
console.log(result)

不要忘记同时启动你的 Milkio 服务器哦！打开任何一个 Milkio 工程中的文件，在 VS Code 左下角点击 ▶️ Milkio 按钮，并选择 Run Milkio & Watch 即可运行。

Don't forget to start your Milkio server at the same time! Open any file in a Milkio project, click on the ▶️ Milkio button in the bottom left corner of VS Code, and select Run Milkio & Watch to run it.
