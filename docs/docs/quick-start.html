快速开始 ​

Quick Start ​

每个故事都会有一场盛大的序幕，使用 Milkio 开始一个全新的工程。

Every story has a grand prologue, start a new project using Milkio.

先决条件 ​

Prerequisites ​

Bun ​

Bun ​

Bun 是 Milkio 的默认运行时。它是一个快速的 JavaScript 运行时，与 Node.js 兼容，并且可以直接运行 TypeScript。

Bun is the default runtime for Milkio. It is a fast JavaScript runtime that is compatible with Node.js and can directly run TypeScript.

bash
curl -fsSL https://bun.sh/install | bash # 适用于 macOS、Linux、WSL
bash
curl -fsSL https://bun.sh/install | bash # for macOS, Linux, and WSL

如果你使用的是 Windows 系统，我建议你使用 WSL2 来进行开发。不用担心 WSL2 会影响你的开发体验，你可以安装 VS Code WSL Extension，像是直接在本机中开发一样方便。

If you are using the Windows operating system, I suggest using WSL2 for development. Don't worry, WSL2 won't affect your development experience. You can install the VS Code WSL Extension to make it as convenient as developing directly on your local machine.

VS Code Extension ​

VS Code Extension ​

你可以前往 此处 安装它。为了做到运行时无关，Milkio 将许多功能整合在了 VS Code Extension 里。在后面的文档中，我们会优先讲述如何通过 VS Code Extension 完成要做的事。

You can go to this link to install it. To achieve runtime independence, Milkio integrates many features in the VS Code Extension. In the following documentation, we will prioritize how to accomplish tasks through the VS Code Extension.

如果你没有 VS Code 编辑器，可以前往 此处 下载它。

If you don't have the VS Code editor, you can go to this link to download it.

安装 ​

Install ​

通过以下命令，可以快速获得一个崭新的 Milkio 工程。其中的 milkio-app 是 Milkio 工程的名称，你可以换成你想要的名字。

You can quickly get a brand new Milkio project with the following command. The milkio-app is the name of the Milkio project, you can replace it with the name you want.

bash
bun create southern-aurora/create-milkio milkio-app
bash
bun create southern-aurora/create-milkio milkio-app

一旦你进入工程目录，你可以安装依赖，然后就可以开始了！

Once you are in the project directory, you can install the dependencies and you're good to go!

cd ./milkio-app
bun install
cd ./milkio-app
bun install

运行你的 Milkio 工程 ​

Run Your Milkio Project ​

使用 VS Code 打开你的 Milkio 工程目录，或者将它添加到你的 VS Code 工作区中也可以。

You can use VS Code to open your Milkio project directory, or you can add it to your VS Code workspace as well.

INFO

此时，你会发现左下角有 ▶️ Milkio 的按钮，点击它，并选择 Run Milkio & Watch 即可运行你的 Milkio 工程。

你可以通过快捷键 (⌘ + ⇧ + L) 或 (Alt + L) 唤出此菜单。

INFO

At this point, you will notice a ▶️ Milkio button in the bottom left corner. Click on it and select Run Milkio & Watch to run your Milkio project.

You can bring up this menu by using the shortcut (⌘ + ⇧ + L) or (Alt + L)

现在，Milkio 已经在 9000 端口中启动！你可以发送一个 POST 请求并带上你的名字，它会帮助你向世界问好～

Now, Milkio is running on port 9000! You can send a POST request with your name, and it will help you say hello to the world~

sh
curl --request POST 'http://localhost:9000/hello-world/say' --data-raw '{"by":"milkio"}'
sh
curl --request POST 'http://localhost:9000/hello-world/say' --data-raw '{"by":"milkio"}'

API 测试 ​

API Test ​

在 VS Code 的资源管理器面板中，打开 src -> apps -> hello-world -> say.ts 你就可以看到它的源代码。

In the VS Code explorer panel, open the src -> apps -> hello-world -> say.ts and you will be able to see its source code.

代码分为两部分，上方是访问 API 时所执行的动作方法，下方是对 API 的测试。你可能注意到了代码中的 test 部分，你可以在上方编写你 API 的逻辑，并在下方编写相应的测试代码。

The code is divided into two parts. The upper part consists of the action methods for accessing the API, while the lower part is for testing the API. You may have noticed the test section in the code. You can write the logic for your API in the upper part and write corresponding test code in the lower part.

ts
export const test = defineApiTest(api, [{
  name: "Basic", // 测试的名称
  handler: async (test) => {
    const result = await test.execute({
      by: "milkio",
    })

    if (!result.success) return test.reject(`The result was not success`)
  },
}])
